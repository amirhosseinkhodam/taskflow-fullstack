# General Documentation

Reusable development guidelines and best practices applicable across all projects. These are not tied to any specific codebase — adapt them to your stack.

## Documents

| File | Description |
|---|---|
| [coding-conventions.md](./coding-conventions.md) | Coding standards — naming, file structure, imports, TypeScript hygiene, component patterns |
| [security-checklist.md](./security-checklist.md) | Security best practices — authentication, authorization, input validation, secrets management |
| [testing-strategy.md](./testing-strategy.md) | Testing approach — unit/integration/e2e strategy, mocking patterns, coverage goals |

## How to Use

1. Copy these files to any project as a starting point
2. Customize the rules to match your team's conventions
3. Reference them in your `AGENTS.md` or coding guidelines
